=========================== WHY NAIVE UNITY FAILS ===========================
Setting UNITY_BUILD ON makes every batch fail: the 145 test files share file-scope
helper NAMES that collide when CMake concatenates them into one unity TU. Two error
classes, both from the same cause:
  - "redefinition of 'X'": kNaN, make_panel, Lcg, PanelFixture, shared_lib,
    same_cell, compile_ok, ts, inst, kCap, kA/kB/kC, make_genome, register_aliases,
    seed_exprs, Fixture, ProbePayload, default function arguments, etc.
  - "call to 'Y' is ambiguous" / "reference to 'Z' is ambiguous": from per-file
    `using namespace ...;` directives leaking into the next file in the batch, which
    then sees two same-named helpers (random_cols, bar_row, make_streams, row, SlotPool).
ANONYMOUS NAMESPACES DO NOT FIX THIS: all files' anonymous namespaces collapse into
the single anonymous namespace of the unity TU, so they still collide (confirmed:
`(anonymous namespace)::Fixture` and `::ProbePayload` clashed).

=========================== THE FIX (per test file) ===========================
Wrap each test file's body in a FILE-UNIQUE NAMED namespace, placed BELOW the
`#include` block. Rule, applied to every atx-engine/tests/*_test.cpp:
  1. Leave ONLY the `#include ...` lines at the top, OUTSIDE the namespace.
     (Including a header inside a namespace is illegal — never wrap includes.)
  2. Open `namespace atxtest_<stem> {` immediately AFTER the last top-of-file
     include, where <stem> = the filename without `.cpp`, already a valid C identifier
     (e.g. backtest_loop_test.cpp -> `namespace atxtest_backtest_loop_test {`).
  3. Put EVERYTHING else inside it: `using` directives/declarations, file-scope
     constants, helper functions, fixtures/structs, and ALL TEST()/TEST_F() macros.
     Putting the `using` directives INSIDE the named namespace is REQUIRED — it stops
     them leaking to the next file in the batch (fixes the "ambiguous" errors).
  4. Close `}  // namespace atxtest_<stem>` at end of file.
  5. If a file already wraps its body in `namespace { ... }`, REPLACE that anonymous
     `namespace {` with the named one (do not nest).
This is valid in BOTH unity and non-unity builds. gtest TEST/TEST_F work inside a
named namespace (registration is runtime + name-string based; the C++ namespace does
not change the "Suite.Name" gtest id). TEST_F fixtures resolve within the same file's
namespace. Do them ALL up front (don't iterate file-by-file) to minimize rebuilds.

Watch for (rare) and handle case-by-case:
  - A file-scope `#define` that another file redefines: a macro leaks across unity
    regardless of namespace. If a "macro redefinition" appears, add `#undef NAME` at
    that file's end, or as a last resort exclude that one file:
    `set_source_files_properties(<file>.cpp PROPERTIES SKIP_UNITY_BUILD_INCLUSION ON)`.
  - A test file that genuinely shares a symbol with another file via a common header:
    that symbol lives in the header, not the .cpp — leave it; only wrap the .cpp's own
    file-local decls. If wrapping breaks a cross-file reference, exclude that file via
    SKIP_UNITY_BUILD_INCLUSION rather than forcing it.

=========================== CMAKE WIRING ===========================
Add a dedicated option so CI stays strict (per-TU include hygiene) and only dev is fast:
  - Root CMakeLists.txt: `option(ATX_UNITY_BUILD "Unity/jumbo build atx test target (dev speed)" OFF)`.
  - CMakePresets.json `dev` preset cacheVariables: add `"ATX_UNITY_BUILD": "ON"`
    (it already sets ATX_USE_PCH ON). The canonical `ninja` preset leaves both OFF.
  - atx-engine/tests/CMakeLists.txt, after the existing ATX_USE_PCH block:
        if(ATX_UNITY_BUILD)
            set_target_properties(atx-engine-tests PROPERTIES
                UNITY_BUILD ON
                UNITY_BUILD_BATCH_SIZE <N>)
        endif()
    Keep PCH ON together with unity (they compose). Tune <N> for 16 cores: 145 files /
    ~16 jumbo TUs ≈ batch 8–10. MEASURE batch sizes {8, 12, 16} and keep the fastest.
  - STRETCH (optional, measure separately): the engine lib (44 src TUs) also re-parses
    Eigen. Do NOT wrap lib .cpp bodies in extra namespaces (their definitions must stay
    in their declared namespaces — wrapping breaks linkage). If you unity the lib, only
    fix its anon-namespace/file-local HELPER clashes (named-namespace just the helper
    section) or exclude offenders; prioritize the 145-TU test target first.
